# 6G-Edge-Offloading-MARL

## 📌 项目背景

随着6G网络的发展，网络架构正在向**分布式、智能化、全服务化**方向演进。在这种背景下，**边缘计算（Edge Computing）**成为降低延迟、提升系统性能的重要技术。

在大规模物联网（IoT）场景中，大量设备需要进行计算任务（如图像识别、数据分析等）。这些设备可以选择：

* 本地计算（延迟高但稳定）
* 卸载到边缘服务器（速度快但可能拥塞）

如何在动态环境下做出最优决策，是一个关键问题。

---

## 🎯 场景描述

本项目模拟一个**智慧城市中的边缘计算场景**：

* 多个IoT设备（Agent）同时产生计算任务
* 多个边缘服务器（Edge Server）提供计算资源
* 所有设备需要自主决定任务执行方式

👉 由于多个设备共享服务器资源，会产生：

* 服务器拥塞
* 延迟增加
* 能耗变化

---

## 🎯 任务目标

本项目的核心任务是：

> **设计一种智能决策方法，使多个设备能够在分布式环境下，实现最优任务卸载策略**

具体目标包括：

* 最小化任务执行延迟
* 降低设备能耗
* 避免服务器过载
* 实现系统整体负载均衡

---

## 🧠 方法介绍

本项目采用：

### 👉 多智能体强化学习（Multi-Agent Reinforcement Learning, MARL）

具体方法为：

* 每个IoT设备作为一个Agent
* 使用 **Q-learning** 学习最优策略
* 通过与环境交互不断优化决策

每个Agent的动作包括：

```text
0 → 本地执行
1~N → 卸载到对应边缘服务器
```

奖励函数综合考虑：

* 延迟（Delay）
* 能耗（Energy）
